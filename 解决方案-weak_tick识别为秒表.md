# weak_tick → 秒表：实测诊断与解决方案

> 对象：`factory_raw/epump2/weak_tick`（11 个 0529 漏判样本，operator 判秒表、模型判 OK）
> 目标类：`秒表 / tick_tock`（`label_rules.yaml` id=102）
> 对照：`key_sns/tick_tock`（40 个确认强秒表）、`2026-05-03` 中 operator 判 OK 的样本
> 配图：`outputs/weak_tick_bandsplit.png`

---

## 结论先行（已用真实数据验证）

1. **现模型对这 11 个 weak_tick 是"零概率"漏判,不是阈值问题。**
   `20260525` 模型对它们的秒表概率(proba_4)几乎全是 0.000–0.008,OK 概率 0.89–1.00。
   说明**特征层根本没看见 tick**,光调阈值/类权重救不回来。

2. **weak_tick 和"强秒表"是两种不同频段的现象——这是漏判的根因。**
   - **强秒表**的冲击能量在**低/中频带(0.5–5 kHz)**:该带"分窗包络峭度"区分
     强秒表 vs OK 的 Cohen's d ≈ **1.0**(图右,绿点有 20–124 的离群冲击)。
   - **weak_tick**在低/中频带几乎没有信号(峭度≈0,比 OK 还低),它的能量在
     **高频带(7–9.5 kHz,接近 Nyquist)**:该带"分窗包络峭度"区分 weak_tick vs OK
     的 d ≈ **1.5,AUC ≈ 0.87**(图左,红点显著抬高),而强秒表在这个高频带反而
     和 OK 一样低。

   → 这是一个干净的**双向分离**:模型主要在强秒表的低/中频带上学到了"秒表",
   而 weak_tick 是高频弱click(人耳听得到、振动低频带几乎没有),所以被判 OK。

3. **所以解决方案的核心是"补高频带 + 时域分窗 + 去瞬态",不是调模型。**
   下面三步是按实测有效性排序的。

---

## 一、为什么现在漏判（实测机理）

- **特征频带偏低。** v6 的 mel / dwt 主轴集中在低/中频,强秒表能被覆盖,
  weak_tick 的 7–9.5 kHz click 没有被任何强特征强调 → 模型看不到。
- **全局统计稀释了局部 click。** weak_tick 的 click 是**时间上局部的短促事件**。
  用整段做统计(全局 period_snr / kurtosis)会被整段噪声稀释——实测全局
  period_snr 对强秒表的 d 只有 0.6,对 weak_tick 是 −0.2(没区分度);
  改成**分窗取最大**后,强秒表低频带 d 立刻升到 1.0、weak_tick 高频带升到 1.5。
- **启停瞬态污染。** 每条录音**前 ~0.25 s 有巨大启停冲击**(幅值 200–900,
  是稳态 std 的 7–25 倍)。若不裁掉,任何峭度/自相关都会被它主导
  (我第一版诊断没裁,选带被瞬态劫持、峭度算出 ~2000 的假值)。**去瞬态是前提。**

---

## 二、解决方案（按实测有效性排序）

### 第 1 步：补"高频带分窗冲击度"特征（直接命中 weak_tick，d≈1.5 / AUC≈0.87）

在 `extract_features_v7.py` 新增,对 **ACC** 通道:

```
预处理:  裁掉首尾各 0.6 s 启停瞬态 → 带通 7–9.5 kHz
分窗:    1 s 窗 / 0.5 s 跳,逐窗算 |hilbert| 包络的峭度
聚合:    取所有窗的最大值  → v7_tick_hi_envkurt_max
配套:    v7_tick_hi_envkurt_p90 / _crest / _shock_count_per_sec(同带分窗)
```

实测:该特征上 64% 的 weak_tick 超过 OK 的 90 分位,AUC 0.87。这是单维就能把
weak_tick 拉出来的特征,现有 220 维里没有等价物。

> 注意频带:7–9.5 kHz 最好(d=1.49),8–9.5 kHz 次之(d=1.68 但召回略低),
> 9 kHz 以上掉得很快(d=0.3,贴 Nyquist 失真)。建议 7–9.5 kHz。

### 第 2 步：保留并强化"低/中频带分窗冲击度"（覆盖强秒表，d≈1.0）

同样的分窗包络峭度,但带通 0.5–5 kHz → `v7_tick_lomid_envkurt_max`。
强秒表在这一带 d≈1.0。**两个带的特征同时加**,模型才能同时覆盖强/弱两类秒表
(它们频段互补,见配图)。

### 第 3 步：多通道 / 上下行融合 + 去瞬态（贯穿前两步）

- **去瞬态**:特征计算前统一裁首尾 0.6 s(现 `process_data` 的 cut_len=8000≈0.4 s
  偏短,瞬态残留;建议提到 0.6 s 或用稳健 clip 压住 >q99.9 的样点)。
- **up/down 取最大**:weak click 常只在一个方向显著,特征层对 up/down 做 max-pool
  比单方向稳。
- **(可选)CURRENT 通道**:实测 CURRENT 低频带分窗峭度对 weak_tick 也有 d≈0.5 的
  弱信号,可作为辅助维(电机电流对机械卡顿的间接反映)。模型当前只用 ACC,
  补 CURRENT 是后续增量方向。

---

## 三、标签与训练侧（特征到位后再做）

1. **weak_tick 并入秒表类并单独建账**:`label_rules.yaml` 的 `tick_tock.alias`
   加 `weak_tick`;把这 11 条(及后续同类)作为**秒表的高频弱样本子集**纳入训练,
   单列召回指标。
2. **类权重 + 弱样本过采样**:沿用 sqrt-balanced,秒表额外上调
   (项目已有 `epump2_general_20260509_top100.yaml` 里 `秒表:2.0` 先例),
   weak 子集再 ×1.5 或轻度过采样(±10% 幅值缩放 + ±0.1 s 平移)。
3. **per-class 阈值 + 双向阈值规则**(v4 文档已建议未落地):秒表用 F1-最优阈,
   并允许"NOK 概率够高时从 OK 翻 NOK"。

> 顺序很重要:**先补第二节的高频特征再训练**。只调权重/阈值救不了"特征零概率"的
> 漏判(第一节已证)。

## 四、落地步骤

1. 跑诊断脚本 `analysis/diagnose_weak_tick.py`(已更新为实测有效版本),在你机器上
   复现"高频带分窗峭度区分 weak_tick vs OK 的 AUC"。
2. 把第二节的两个带(7–9.5 kHz / 0.5–5 kHz)分窗冲击度特征落 `extract_features_v7.py`

3. weak_tick 并入秒表标签,重训,单看 weak_tick recall 与整体 false_ok_rate。

## 五、预处理"冲击增强"对比（实测，回答"能不能先增强再判"）

在真实 weak_tick(11) vs OK(30) 上,对 6 条常见冲击增强预处理链做了 AUC/d 对比
(脚本 `analysis/enhance_preprocess_test.py`):

| 预处理链 | Cohen's d | AUC | 召回@OK-p90 |
|---|---:|---:|---:|
| **0 基线**(裁瞬态→带通7–9.5k→Hilbert包络→分窗峭度max) | **1.22** | **0.83** | 36% |
| 1 谱白化(magnitude 拉平) | 1.16 | 0.81 | **45%** |
| 6 crest factor(峰值/RMS) | 1.03 | 0.82 | 27% |
| 2 倒谱预白化 CPW | 0.96 | 0.75 | 27% |
| 4 Teager-Kaiser TKEO | 0.41 | 0.63 | 27% |
| 3 AR 残差白化 | 0.27 | 0.64 | 9% |
| 5 MED 反卷积 | −0.36 | 0.48 | 0% |

**结论:复杂预处理都没赢过基线。** 真正有效的"冲击增强"就是**去启停瞬态 +
选高频带(7–9.5 kHz)**本身;在此之上再做白化 / MED / TKEO 不仅无益,大多变差。

原因:7–9.5 kHz 高频带里几乎没有泵转频那类稳态谐波,"去背景"类方法(CPW/AR残差)
无背景可去、只放大噪声;MED/TKEO 把 OK 样本的噪声尖峰也一起放大(MED 的 d=−0.36、
峭度飙到 ~2200 即过白化放大噪声),反而拉低类间区分。

**可选**:谱白化把召回@p90 从 36% 提到 45%(但 d/AUC 略降),可作为**辅助维**与基线
并用,不替代基线。

> 注:weak 只有 11 条,基线召回@OK-p90 仅 ~36%,说明单一特征是"部分提升",真正达标
> 仍需多特征 + 第三/四节的标签与阈值工作配合。

## 六、实测证据汇总

样本:weak_tick=11,strong tick_tock=40,OK(0529 operator 判 OK)=60。特征=分窗
(1 s/0.5 s)包络峭度的最大值,up/down 取 max,裁首尾 0.6 s。

| 频带 | 特征 | d(weak−OK) | d(strong−OK) | 含义 |
|---|---|---:|---:|---|
| 7–9.5 kHz(高) | 分窗包络峭度 max | **+1.49 (AUC 0.87)** | +0.09 | **weak_tick 在这里** |
| 0.5–5 kHz(低/中) | 分窗包络峭度 max | −0.15 | **+1.02** | **强秒表在这里** |
| 全段 period_snr(对照) | 自相关周期 SNR | −0.20 | +0.60 | 全局统计被稀释,弱 |

模型现状(`20260525`)对 11 个 weak_tick:秒表概率 0.000–0.008,全部判 OK。

**一句话**:weak_tick 是**高频(7–9.5 kHz)、时间局部、振动低频带几乎不可见的弱
click**;补"高频带分窗冲击度"特征 + 去瞬态 + 上下行融合,是把它从 OK 里救出来的
关键,且不影响现有强秒表的低/中频带特征。
